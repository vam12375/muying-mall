## 5.4 商品与搜索模块实现

商品搜索模块采用 **Spring Data Elasticsearch + Elasticsearch Java Client 9.2.1** 实现。  
其中，索引文档模型由 `ProductDocument` 定义，查询逻辑由 `ProductSearchServiceImpl` 实现，控制层通过 `/search/*` 接口对外提供搜索、联想词、聚合筛选等能力。

### 5.4.1 倒排索引构建

系统使用 `ProductDocument` 作为商品搜索文档实体，并通过 `@Document(indexName = "products", createIndex = false)` 绑定到 `products` 索引。

在字段设计上，`productName` 采用 `@MultiField` 配置：

- 主字段类型为 `Text`，索引分词器为 `ik_max_word`，查询分词器为 `ik_smart`。
- 同时提供 `productName.keyword` 子字段，用于精确匹配和精确排序场景。

此外，`productName` 上定义了 `@JsonProperty("name")` 以兼容 JSON 序列化字段名；但在搜索查询构建中实际使用的字段仍是 `productName`。  
同类中文检索字段（如 `productDetail`、`categoryName`、`brandName`、`keywords`）也采用了 `ik_max_word/ik_smart` 的分词组合。

### 5.4.2 复杂查询逻辑实现

在 `ProductSearchServiceImpl#searchProducts` 中，系统基于 Elasticsearch Java Client 原生 API 构建 `BoolQuery` 复合查询，主要包含以下能力：

1. 关键词匹配：当关键词长度小于 3 或包含中文时，使用 `MultiMatch` 精确召回策略（不启用模糊）；其他场景启用 `fuzziness("AUTO")`，并设置 `prefixLength(2)`、`maxExpansions(50)`；多字段加权检索字段为 `productName^3`、`productDetail^2`、`categoryName^1.5`、`brandName^1.5`、`keywords^1`。  
2. 条件筛选：支持 `categoryId`、`brandId` 的 `term` 精确过滤；支持 `productPrice` 区间过滤；固定过滤 `productStatus = "上架"`。  
3. 排序策略：显式排序字段支持 `price`、`sales`、`rating`、`createTime`（支持升降序）；未指定排序时，默认按 `score desc + searchWeight desc + salesCount desc` 复合排序。  
4. 查询结果优化：通过 `_source excludes` 排除 `productDetail`、`specs` 大字段以减少传输开销；启用 `productName` 与 `productDetail` 高亮；支持分页参数 `from(page * size)` 与 `size(size)`。  
5. 容灾降级：Elasticsearch 查询异常时降级至数据库查询；Elasticsearch 返回空结果且存在关键词时也会尝试数据库兜底；数据库兜底使用 MyBatis-Plus `LIKE`（`productName` / `productDetail`）并复用分类、品牌、价格与排序条件。  

核心实现示例代码如下：

```java
BoolQuery.Builder boolQueryBuilder = new BoolQuery.Builder();

if (StringUtils.hasText(keyword)) {
    Query keywordQuery = buildOptimizedKeywordQuery(keyword);
    boolQueryBuilder.must(keywordQuery);
}

if (categoryId != null) {
    boolQueryBuilder.filter(Query.of(q -> q.term(t -> t.field("categoryId").value(categoryId))));
}
if (brandId != null) {
    boolQueryBuilder.filter(Query.of(q -> q.term(t -> t.field("brandId").value(brandId))));
}
if (minPrice != null || maxPrice != null) {
    boolQueryBuilder.filter(Query.of(q -> q.range(r -> r.number(n -> {
        n.field("productPrice");
        if (minPrice != null) n.gte(minPrice.doubleValue());
        if (maxPrice != null) n.lte(maxPrice.doubleValue());
        return n;
    }))));
}
boolQueryBuilder.filter(Query.of(q -> q.term(t -> t.field("productStatus").value("上架"))));

SearchRequest searchRequest = SearchRequest.of(s -> s
    .index("products")
    .query(Query.of(q -> q.bool(boolQueryBuilder.build())))
    .sort(buildSortOptions(sortBy, sortOrder))
    .from(page * size).size(size)
    .source(src -> src.filter(f -> f.excludes("productDetail", "specs")))
    .highlight(h -> h.fields(
        co.elastic.clients.util.NamedValue.of("productName",
            co.elastic.clients.elasticsearch.core.search.HighlightField.of(hf -> hf.preTags("<em>").postTags("</em>"))),
        co.elastic.clients.util.NamedValue.of("productDetail",
            co.elastic.clients.elasticsearch.core.search.HighlightField.of(hf -> hf.preTags("<em>").postTags("</em>")))
    ))
);

SearchResponse<ProductDocument> response =
    elasticsearchClient.search(searchRequest, ProductDocument.class);
```

补充说明：后端已提供 `/search/suggestions` 与 `/search/aggregations` 接口，前端可据此实现关键词联想与品牌/分类/价格区间筛选。
